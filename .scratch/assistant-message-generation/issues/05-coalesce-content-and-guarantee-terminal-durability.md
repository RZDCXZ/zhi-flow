# 05 — 合并正文持久化并保证终态耐久性

**What to build:** 让用户继续及时看到流式正文，同时减少逐 delta 数据库写入。正文按明确上限合并持久化，任何终态事件都必须由完整正文和数据库原子终态证明；持久化不可用时不发送伪终态。

**Blocked by:** 04 — 让生成生命周期独立于 SSE 订阅

**Status:** resolved

- [x] 正文事件不等待每个 delta 的数据库写入即可继续产生。
- [x] 累计 500 ms 或 1,024 个字符任一先到时刷新当前完整正文。
- [x] 进入 `completed`、`cancelled` 或 `failed` 前强制刷新全部已知正文。
- [x] 只有完整正文和原子终态成功提交后，订阅者才能观察对应终态事件。
- [x] 最终正文或终态提交失败时，事件订阅以 transport interruption 结束，不产生无法由数据库证明的 `message.failed`。
- [x] 进程意外终止时，尚未持久化的正文尾段被限制在已声明的时间或大小上限内。
- [x] 持久化配置留在服务端，不扩大 module 的每次调用 interface。
- [x] 可控时间测试覆盖时间阈值、大小阈值、终态强制刷新、部分正文失败和持久化不可用。
