# 18 长对话摘要与 Token 预算

Type: task
Status: ready-for-agent
Blocked by: 17

## 学习目标

理解上下文窗口不是数据库，滚动摘要、最近 12 条 Message、检索证据和输出预算如何共同受限。

## 前置依赖

- 里程碑 17 已完成并获准继续。

## 最小实现

- Conversation 保存滚动摘要和摘要覆盖到的最后 Message。
- 默认保留最近 12 条已完成 Message，较早内容增量摘要；摘要失败时保留旧摘要。
- 建立模型上下文与最大输出的配置预算，按安全指令、摘要/近期消息、证据优先级裁剪并记录排除项。

## 运行观察

- 生成超过窗口阈值的对话，查看何时摘要、摘要覆盖范围、Token 分配和证据裁剪。
- 模拟摘要超时、重复触发和并发请求，观察是否重复总结或丢历史。

## 验证方法

- API 接缝验证摘要边界、最近 12 条、失败降级、并发一致性和预算不超限。
- 使用假 Token 计数器构造确定边界，不在测试中依赖真实模型 tokenizer 波动。

## 完成标准

- 长会话请求始终在配置预算内；摘要不重复覆盖、不混入未完成 Message；关键近期语境仍可用。
- 展示上下文预算，等待用户决定是否进入里程碑 19。

## 本阶段暂不处理

- 语义记忆、跨会话长期记忆、Prompt 缓存和自动模型切换。
