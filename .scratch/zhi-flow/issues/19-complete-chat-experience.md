# 19 完整聊天体验与消息状态机

Type: task
Status: ready-for-agent
Blocked by: 18

## 学习目标

把流式、持久化、停止、失败、重生成、重复请求和界面恢复整合为明确的 Message 状态机。

## 前置依赖

- 里程碑 18 已完成并获准继续。

## 最小实现

- 完整实现助手 `streaming`、`completed`、`cancelled`、`failed` 的允许迁移与终态保护。
- 停止按助手 Message 传播取消；重新生成对同一用户 Message 建立新尝试且保留旧尝试。
- 完成 Conversation/Knowledge Base 左栏、聊天主区、错误与重试体验，以及窄屏抽屉布局。
- 断线或刷新后以数据库终态恢复，不自动重复模型副作用。

## 运行观察

- 快速重复发送、停止后重生成、两个标签页、断线重连和旧 `streaming` 恢复。
- 对比旧尝试与新尝试在历史、引用和上下文中的选择规则。

## 验证方法

- HTTP/SSE 接缝覆盖所有合法/非法状态迁移、幂等和并发。
- 浏览器冒烟覆盖发送、流式、停止、重生成、刷新和移动窄屏基本操作。

## 完成标准

- UI 与数据库状态一致；每个请求只有一个终态；重生成不覆盖历史或重复用户 Message。
- 展示状态机实验，等待用户决定是否进入里程碑 20。

## 本阶段暂不处理

- 多人实时协作、离线编辑、语音、复杂动画和完整无障碍专项审计。
