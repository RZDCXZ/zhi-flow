# 03 非流式单轮聊天

Type: task
Status: ready-for-agent
Blocked by: 02

## 学习目标

理解一次 LLM 请求的消息结构、后端配置、薄 Chat Provider、超时、错误分类与用量记录。

## 前置依赖

- 里程碑 2 已完成并获准继续。

## 最小实现

- 定义 Chat Provider 合约及可控假 Provider。
- 实现一个仅服务端可调用的非流式单轮聊天 API 与极简输入/答案界面。
- 后端读取 OpenAI-compatible 配置，验证输入长度，返回脱敏错误、延迟与 Token 用量。

## 运行观察

- 比较假 Provider 与真实 Provider 的请求、响应、延迟和错误。
- 观察超时、401、429 和 5xx 如何映射为稳定应用错误。

## 验证方法

- 通过公开 HTTP 接缝测试成功、空输入、超长输入、超时和供应商错误。
- 真实模型只做显式手动 smoke，不进入普通自动测试。

## 完成标准

- 单轮请求可靠返回；模型信息和密钥不出现在客户端；Provider 合约测试通过。
- 展示调用链和失败路径，等待用户决定是否进入里程碑 4。

## 本阶段暂不处理

- 流式输出、持久化、多轮上下文、RAG 和供应商切换 UI。
