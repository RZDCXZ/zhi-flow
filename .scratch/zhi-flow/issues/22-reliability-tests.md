# 22 可靠性测试

Type: task
Status: ready-for-agent
Blocked by: 21

## 学习目标

系统验证超时、限流、流中断、重复请求、损坏文件和队列故障下的终态、幂等与可恢复性。

## 前置依赖

- 里程碑 21 已完成并获准继续。

## 最小实现

- 建立故障注入矩阵：聊天首字节/空闲/总超时、429/5xx、流中断、取消竞争、重复幂等键、队列租约过期、Consumer 崩溃、损坏文件、向量错维、空召回、伪造 Citation 和删除传播。
- 统一稳定错误码、可重试分类、脱敏日志与请求关联 ID。
- 验证非流式调用最多 3 次重试、流正文开始后不自动重启、Consumer 最多 5 次后归档。

## 运行观察

- 逐一触发故障，查看 UI、数据库、队列、日志和 RAG Run 是否讲述同一个结果。
- 恢复外部依赖后重试，确认没有重复 Message、Chunk 或 Citation。

## 验证方法

- 主要在 HTTP/SSE 和 Consumer 两个高层接缝执行自动故障矩阵。
- 少量浏览器冒烟验证用户能理解并执行停止、重试或重新上传。

## 完成标准

- 每个故障都有稳定终态、无秘密泄露、无静默丢失，并明确是否可重试；矩阵全部通过。
- 展示最关键的故障恢复，等待用户决定是否进入里程碑 23。

## 本阶段暂不处理

- 多区域灾备、自动熔断平台、SLO 告警系统和混沌工程基础设施。
