# 20 RAG 调试视图

Type: task
Status: ready-for-agent
Blocked by: 19

## 学习目标

把一次 RAG 失败分解到问题改写、召回、重排、上下文、生成和 Citation 校验阶段。

## 前置依赖

- 里程碑 19 已完成并获准继续。

## 最小实现

- 持久化并展示 RAG Run：原问题、独立问题、召回 20、重排前后、Top-5 上下文、阈值、阶段耗时、Token 和错误码。
- 右侧调试抽屉默认折叠，与面向用户的 Citation 视图分区。
- 对普通聊天显示“未使用 RAG”，不返回 Prompt 全文、密钥或未脱敏供应商异常。

## 运行观察

- 选择检索失败、重排改善、阈值拒答、生成失败和 Citation 校验失败案例逐层查看。
- 对比 RAG Run 配置快照，确认之后仍可解释当时结果。

## 验证方法

- API 接缝断言阶段数据与实际候选/终态一致，敏感字段不在响应中。
- 浏览器冒烟验证抽屉、排序对比、Citation 联动和普通聊天空状态。

## 完成标准

- 用户能从调试视图定位代表性失败阶段；显示数据可追踪且不泄密。
- 展示一条成功和多条失败 Trace，等待用户决定是否进入里程碑 21。

## 本阶段暂不处理

- 分布式追踪平台、生产 Dashboard、Prompt 全文回放和在线反馈系统。
