# 17 多轮 RAG 与问题改写

Type: task
Status: ready-for-agent
Blocked by: 16

## 学习目标

理解追问为何需要改写为独立问题，以及改写上下文、检索查询和最终回答上下文应分别控制。

## 前置依赖

- 里程碑 16 已完成并获准继续。

## 最小实现

- 在检索前用已完成的最近 Conversation 历史把追问改写为独立问题。
- 改写 Provider 只返回问题，不回答；失败时采用明确错误或保守原问题降级并记录原因。
- RAG Run 保存原问题、独立问题和对应检索结果；调试区先显示改写文本。

## 运行观察

- 测试代词、省略主语、话题切换、纠正前问和历史中的提示注入。
- 比较直接检索追问与改写后检索的候选差异。

## 验证方法

- HTTP/SSE 接缝使用记录输入的假 Provider 断言改写顺序、Conversation 隔离和检索使用独立问题。
- 固定多轮夹具验证能命中预期 Chunk，失败改写不跨会话取历史。

## 完成标准

- 代表性追问可稳定改写并提升或保持检索；原问题与独立问题均可追踪。
- 展示多轮对比，等待用户决定是否进入里程碑 18。

## 本阶段暂不处理

- 长对话摘要、自动多查询、Agent 记忆和跨 Conversation 记忆。
