# 01 项目骨架

Type: task
Status: ready-for-agent

## 学习目标

理解 Next.js 全栈应用、环境配置、服务端/客户端边界、质量命令和本地开发入口如何组成最小工程。

## 前置依赖

- 规格已获用户确认。

## 最小实现

- 初始化 Next.js、TypeScript、Tailwind CSS 与 shadcn/ui。
- 建立统一的后端配置读取与启动时校验；提供不含真实密钥的示例配置。
- 建立格式化、静态检查、类型检查和测试命令。
- 创建只证明服务可运行的首页与健康检查，不实现聊天或数据库业务。

## 运行观察

- 对比浏览器组件与服务端健康检查的运行位置。
- 故意缺失一个必需配置，观察启动失败是否清楚且不泄密。

## 验证方法

- 运行质量命令与生产构建。
- 从浏览器和 HTTP 客户端验证首页、健康检查及错误配置行为。

## 完成标准

- 新环境按说明可启动；所有质量命令通过；客户端产物不包含服务端密钥或模型配置。
- 展示运行结果并解释模块边界，等待用户决定是否进入里程碑 2。

## 本阶段暂不处理

- 数据库、聊天调用、文件上传、部署和完整产品布局。
